Gong-Verse Data Collection

Why

The Gong-Verse Data Collection project was initiated to build a robust system for collecting, validating, and storing large amounts of data from international clients. With over 60,000 data entries to process, the project required a solution that could efficiently handle data validation while maintaining accuracy and integrity. Certain parts of the project had to be kept secret and git-ignored such as the validation process against company data to ensure client data safety

How

I designed and developed a full-stack custom web application with a Node.js backend that integrated with Google Sheets API for persistent data storage and retrieval. The application includes:

  • Data validation against company records to ensure accuracy before acceptance
  • Selective parsing of relevant fields to optimize processing speed
  • Caching of previous inputs to reduce redundant operations
  • Batched column reads for improved performance
  • Responsive frontend built with HTML and CSS for a smooth user experience

What

The final result is a performant web application that successfully processes over 60,000 data entries from international clients. The system allows for efficient data collection while ensuring data integrity through validation against company records. The application's architecture optimizes performance through strategic data processing techniques and provides a user-friendly interface for data submission.

Technologies used: JavaScript, Node.js, Google APIs, HTML, CSS, and deployed on Render.
